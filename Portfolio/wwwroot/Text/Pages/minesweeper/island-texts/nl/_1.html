Input van de keyboard was een van de dingen die ik al snel moeilijk vond worden. Voornamelijk ook omdat de manier van het uitlezen van input voor een Linux-systeem zo compleet anders is dan dat voor een Windows-systeem.

Voor een Windows-systeem is er een ingebouwde API uit de conio.h header. Deze API, _getch , bied een mogelijkheid om directe keyboard input uit te lezen zonder dat dit te zien is in de console zelf. In het geval we het hexidecimale getal 0xE0 uitlezen, verwachten we nog een "input", dit is het geval voor meeste non-alphanumeric keys van de keyboard.

Op een Linux-systeem werkt dit net wat anders. Ten eerste moet input mode op Non-Canonical staan en moet de ECHO naar de terminal uit staan. En om de input uit te lezen, lees ik 8 bytes uit de file-descriptor van stdin (dit is zodat ik non-blocking kan uitlezen). Ook return ik hoe hoog de "degree" is die ik heb uitgelezen, welke de hoeveelheid aan uitgelezen bytes aangeeft.
